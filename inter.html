Теория API-документации
Выберите правильный ответ на каждый вопрос. Изучите теорию перед ответом.
📚 ТЕОРИЯ OpenAPI (Swagger) — стандарт описания REST API в формате YAML/JSON. Файл содержит: эндпоинты, методы HTTP, параметры запроса, форматы ответов и схемы аутентификации.

Ключевые инструменты документирования:
• Swagger UI — интерактивная документация с возможностью тест-запросов
• Redoc — красивая статичная документация из OpenAPI-файла
• Postman — документирование + тестирование API-коллекций
• Javadoc / KDoc — документирование кода Java/Kotlin (тегами /** */)

HTTP-методы: GET (получить), POST (создать), PUT (заменить), PATCH (обновить), DELETE (удалить)
02
Структура OpenAPI-спецификации
Расставьте секции OpenAPI 3.0 файла в правильном порядке перетаскиванием
📚 ТЕОРИЯ Файл OpenAPI 3.0 состоит из обязательных секций в определённом логическом порядке:

openapi → версия спецификации (например, "3.0.3")
info → метаданные API (название, версия, описание, контакты)
servers → список базовых URL серверов (production, staging, dev)
paths → эндпоинты и операции (GET /users, POST /auth и т.д.)
components → переиспользуемые схемы, параметры, ответы
security → глобальные требования аутентификации

Перетащите блоки в правильном порядке (сверху вниз):

⠿?
paths: — эндпоинты и HTTP-операции
⠿?
openapi: — версия спецификации
⠿?
security: — требования аутентификации
⠿?
servers: — список URL-серверов
⠿?
info: — метаданные API
⠿?
components: — переиспользуемые схемы
03
Заполни OpenAPI-документацию
Вставьте правильные ключевые слова в YAML-спецификацию
📚 ТЕОРИЯ operationId — уникальный идентификатор операции
summary — краткое описание операции
parameters — список параметров (path, query, header)
responses — возможные ответы по HTTP-кодам
schema — описание структуры данных (тип, свойства)
paths:
  /users/{id}:
    get:
      : "getUserById"
      : "Получить пользователя"
      :
        - name: id
          in: path
          required: true
      :
        200:
          description: "Успешно"
          content:
            application/json:
              :
                $ref: '#/components/schemas/User'

Слова для вставки: operationId, summary, parameters, responses, schema

📚 ТЕОРИЯ Инструменты документирования API:

• mermaid.js — диаграммы последовательностей для API-флоу

Анализ API-документации
Выберите ВСЕ правильные утверждения для каждого вопроса
📚 ТЕОРИЯ Лучшие практики документирования API:

✅ Описывайте все возможные HTTP-коды ответа (200, 201, 400, 401, 404, 500)
✅ Используйте реальные примеры запросов и ответов
✅ Документируйте аутентификацию (Bearer Token, API Key, OAuth2)
✅ Описывайте ограничения rate limiting (лимиты запросов в секунду)
✅ Версионирование: /api/v1/, /api/v2/ с changelog изменений
❌ Не оставляйте эндпоинты без описания
❌ Не используйте внутренние названия/жаргон без объяснения
