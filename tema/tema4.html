Квиз: основы MongoDB
Выберите правильный ответ. Читайте теорию перед ответом.
📚 ТЕОРИЯ MongoDB — документно-ориентированная NoSQL база данных. Данные хранятся в формате BSON (Binary JSON) — бинарном расширении JSON.

Основные понятия MongoDB:
  • Database — база данных, контейнер коллекций
  • Collection — аналог таблицы в SQL, набор документов
  • Document — запись в формате BSON (аналог строки в SQL)
  • Field — поле документа (аналог столбца)
  • _id — уникальный идентификатор документа, генерируется автоматически как ObjectId

MongoDB Atlas — облачный хостинг MongoDB от разработчиков. Поддерживает кластеры, резервное копирование, глобальные регионы.

MongoDB Realm (Atlas Device SDK) — SDK для мобильных приложений: локальная БД на устройстве + автоматическая синхронизация с Atlas в облаке.

📚 ТЕОРИЯ CRUD — четыре базовые операции с данными:

Create — вставка документов: insertOne() / insertMany()
Read — чтение и поиск: findOne() / find() с фильтрами
Update — обновление: updateOne() / updateMany() с оператором $set
Delete — удаление: deleteOne() / deleteMany()

Порядок типичного запроса чтения (Read) в мобильном приложении:
1. Получить экземпляр клиента MongoDB
2. Обратиться к нужной базе данных
3. Выбрать коллекцию
4. Выполнить запрос с фильтром
5. Обработать результат в UI

Вставь код: запрос к MongoDB Atlas
Заполни пропуски в коде правильными словами
📚 ТЕОРИЯ MongoDB Atlas Device SDK (Realm) для Android/Kotlin:

• Подключение к Atlas через App с appId из консоли
• Аутентификация: Credentials.anonymous(), emailPassword(), google()
• Вход: app.login(credentials) возвращает User
• Доступ к MongoDB: user.getMongoClient("mongodb-atlas")
• Коллекция: db.getCollection<T>("name")
• Вставка: collection.insertOne(document)
• Поиск: collection.findOne(filter)
• Фильтр строится через Document("field", value)
💡 Слова для вставки: login, getMongoClient, getCollection, insertOne, findOne
// Инициализация App
val app = App.create("your-app-id")
val user = app.(Credentials.anonymous())

// Получение клиента MongoDB
val client = user.("mongodb-atlas")
val db = client.getDatabase("myDatabase")

// Получение коллекции
val collection = db.<Document>("users")

// Вставка документа
collection.(Document("name", "Alice"))

// Поиск одного документа
val result = collection.(Document("name", "Alice"))

Нажмите на оператор, затем на его описание. Найдите все 6 пар.
📚 ТЕОРИЯ Операторы запросов и обновлений MongoDB:
  • $set — установить значение поля (при update)
  • $unset — удалить поле из документа
  • $gt — больше чем (greater than) — для фильтрации
  • $lt — меньше чем (less than) — для фильтрации
  • $in — значение из списка: { field: { $in: [v1, v2] } }
  • $push — добавить элемент в массив поля

Пример: updateOne(filter, Document("\$set", Document("age", 25)))

Мультивыбор: MongoDB в мобильной разработке
Выберите ВСЕ верные утверждения в каждом вопросе
📚 ТЕОРИЯ Ключевые факты о MongoDB в мобильных приложениях:

✅ MongoDB Atlas — облачный managed-сервис MongoDB с поддержкой кластеров
✅ Atlas Device SDK (Realm) — обеспечивает локальное хранилище + синхронизацию с облаком
✅ Документы хранятся в формате BSON — бинарном расширении JSON
✅ Поле _id уникально идентифицирует каждый документ (тип ObjectId)
✅ Индексы в MongoDB ускоряют поиск, но замедляют запись
✅ Realm позволяет работать с данными офлайн с последующей синхронизацией
❌ MongoDB НЕ поддерживает JOIN как в SQL — денормализация вместо объединений
❌ ObjectId — это НЕ обычное целое число, это 12-байтный идентификатор
❌ Коллекция в MongoDB НЕ требует одинаковой структуры у всех документов
