01 — Квиз: SQL vs NoSQL и Firebase

📚 ТЕОРИЯ NoSQL (Not Only SQL) — класс СУБД, не использующих реляционную модель. Данные хранятся не в таблицах, а в документах, парах ключ-значение, графах или колонках.

Типы NoSQL баз данных:
  • Document Store — документы JSON/BSON: MongoDB, Firestore
  • Key-Value — быстрый доступ по ключу: Redis, DynamoDB
  • Column-family — данные по колонкам: Cassandra, HBase
  • Graph DB — узлы и рёбра-связи: Neo4j

Firebase — платформа Google с двумя БД:
• Realtime Database — одно большое JSON-дерево, синхронизация в реальном времени
• Firestore — документно-коллекционная модель, мощные запросы, офлайн-кэш

Ключевые отличия от SQL: нет жёсткой схемы (schemaless), горизонтальное масштабирование (sharding), модель BASE вместо ACID.

02 — Сортировка: иерархия Firestore

📚 ТЕОРИЯ Модель данных Firestore строго иерархична. Каждый уровень вложен в предыдущий:

Database → корень всей базы данных
  ↳ Collection — именованный контейнер документов (например, users)
    ↳ Document — единица хранения, JSON-объект с полями (например, users/uid123)
      ↳ Field — конкретное значение внутри документа (например, name: "Alice")

Документ может дополнительно содержать Subcollection — коллекцию внутри самого документа.

Перетащи блоки — сверху самый высокий уровень иерархии:

⠿?

⠿?

⠿?
Database — корневой уровень Firestore
⠿?

03 — Вставь код: инициализация Firebase

📚 ТЕОРИЯ Подключение Firebase к Android-проекту:
  • Создать проект в Firebase Console и добавить Android-приложение
  • Скачать google-services.json и положить в папку app/
  • В build.gradle (project) — подключить classpath плагина Google Services
  • В build.gradle (app) — применить плагин через apply plugin
  • Добавить зависимости через implementation, используя Firebase BoM (Bill of Materials) для единого управления версиями
  • Получить экземпляр: FirebaseFirestore.getInstance()
💡 Слова для вставки: apply, implementation, FirebaseFirestore, google-services
// build.gradle (app)
plugin: 'com.google.gms.google-services'

dependencies {
   platform('com.google.firebase:firebase-bom:32.0.0')
  implementation 'com.google.firebase:firebase-firestore'
}

// В коде приложения (Kotlin)
val db = .getInstance()

// Конфигурационный файл (папка app/)
".json"

04 — Сопоставь инструмент с описанием

📚 ТЕОРИЯ Экосистема Firebase и ключевые понятия NoSQL:
  • Firestore — документная БД с коллекциями, подколлекциями, офлайн-кэшем
  • Realtime Database — JSON-дерево, минимальная задержка, идеально для чатов
  • Firebase Auth — готовая аутентификация: Google, Email, телефон, аноним
  • Cloud Storage — хранение бинарных файлов: фото, видео, документы
  • Firebase Rules — декларативные правила безопасности без серверного кода
  • Sharding — горизонтальное масштабирование: данные делятся между серверами

05 — Мультивыбор: правда о NoSQL и Firebase

📚 ТЕОРИЯ Важные характеристики и распространённые заблуждения:

✅ Firestore поддерживает офлайн-режим — данные кэшируются локально на устройстве
✅ Realtime Database хранит всё в виде одного большого JSON-дерева
✅ Firestore поддерживает составные индексы для сложных запросов
✅ Firebase автоматически синхронизирует данные между всеми подключёнными клиентами
✅ NoSQL хорошо масштабируется горизонтально через шардирование
❌ Firestore НЕ поддерживает JOIN-запросы как в реляционных БД
❌ NoSQL не всегда быстрее SQL — зависит от задачи и структуры запросов
❌ Realtime Database и Firestore — разные продукты с разными моделями данных

06 — Блок-схема

Визуальная схема процесса оценки производительности мобильного приложения

🖼️
Изображение не найдено.
Открой HTML в редакторе и замени путь в теге
<img src="...">
Например: src="blokschema.png"
Файл должен лежать рядом с этим HTML или в указанной папке
💡 Как вставить блок-схему:
1. Положи файл изображения в ту же папку, что и этот HTML
2. Открой файл в редакторе (VS Code, Notepad++)
3. Найди строку src="schemas/image3.png"
4. Замени на имя своего файла: src="моя_схема.png"
5. Сохрани и обнови страницу в браузере (F5)
